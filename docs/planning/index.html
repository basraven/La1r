    Planning

    What's next?

    Since I'm (currently) only developing la1r by myself, there are only so many things you can do at once. (Feel free to reach out through Github if you want to get involved!) For this reason I created this planning page in which I track and prioritize what I will add to la1r next. Feel free to add comments on this through Github!

    In progress

    1. Traefik auth proxy middleware with Authelia

    Planned

    1. Deploy a Kafka bus as structured event bus instead of mqtt
    2. Spark 2.x Cluster in k8s
    3. Streaming analytics pipeline with Spark 2.x and Kafka
    4. An event data dashboard for kafka
    5. Streaming Facial recognition from images and streaming video
    6. Object recognition (garbage bin outside of our house) combined with garbage collection iCal (https://inzamelkalender.gad.nl/ical/0402200001574396)
    7. Formal managed bare-metal security camera setup
    8. “View in repo” button for all pages of the la1r documentation. While reading documentation, for example about Ansible, the visitor should be able to view which scripts are currently discussed by clicking a button to the git repository.
    9. Refactor Mosquito to vernemq

    Parked

    1. Ceph file system based rook.io or on this guide: https://owncloud.org/news/running-owncloud-in-kubernetes-with-rook-ceph-storage-step-by-step/

    Completed

    1. Refactor backup facilities to use compression and dual backup (weekly and bi-daily)
    2. OpenLDAP implementation
    3. Formal service bus definition and separation of raw data and defined event data in MQTT
    4. find-lf - wifi tracking as input events on the Event Bus based on AI models
