                Planning

                What's next?

                Since I'm (currently) only developing la1r by myself, there are only so many things you can do at once. (Feel free to reach out through Github if you want to get involved!) For this reason I created this planning page in which I track and prioritize what I will add to la1r next. Feel free to add comments on this through Github!

                In progress

                1. Spark 2.x Cluster in k8s
                2. Streaming analyitics pipeline with Spark 2.x and Kafka

                Planned

                1. Traefik auth proxy middleware with Authelia
                2. Streaming Facial recognition from images and streaming video
                3. Object recognition (garbage bin outside of our house) combined with garbage collection ical (https://inzamelkalender.gad.nl/ical/0402200001574396)
                4. Formal managed bare-metal security camera setup
                5. “View in repo” button for all pages of the la1r documentation. While reading documentation, for example about Ansible, the visitor should be able to view which scripts are currently discussed by clicking a button to the git repository.
                6. Refactor mosquitto to vernemq

                Parked

                1. Deploy a Kafka bus as structured event bus instead of mqtt
                  • Parked due to server resources shortage
                2. An event data dashboard for kafka
                3. Ceph file system based rook.io or on this guide: https://owncloud.org/news/running-owncloud-in-kubernetes-with-rook-ceph-storage-step-by-step/

                Completed

                1. OpenLDAP implementation
                2. Formal service bus definition and separation of raw data and defined event data in MQTT
                3. find-lf - wifi tracking as input events on the Event Bus based on AI models
